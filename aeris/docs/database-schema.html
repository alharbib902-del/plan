04 · CRITICAL FLOWS

التدفقات الحرجة

✈️ تدفق حجز Charter

1️⃣ Client fills /charter/new form
→ INSERT INTO trip_requests (status='pending')
2️⃣ Auto-distribution engine runs
→ UPDATE trip_requests SET distributed_to = [operator_ids], status='distributed'
→ INSERT INTO notifications (×5 operators, channel='email')
3️⃣ Operators submit offers
→ INSERT INTO offers (status='pending', match_score)
4️⃣ Client selects offer
→ UPDATE offers SET status='accepted'
→ INSERT INTO bookings (payment_status='pending')
5️⃣ Client picks add-ons
→ INSERT INTO booking_addons (×N)
→ UPDATE bookings SET addons_amount, total_amount
6️⃣ Payment via HyperPay
→ INSERT INTO payments (status='initiated')
→ Webhook: UPDATE payments SET status='success'
→ UPDATE bookings SET payment_status='paid'
7️⃣ Post-payment automation
→ INSERT INTO loyalty_transactions (points earned)
→ UPDATE users SET loyalty_points += X, total_spent_yearly += amount
→ Generate ZATCA invoice → UPDATE bookings SET zatca_*
→ INSERT INTO notifications (email + sms + whatsapp link)
8️⃣ Post-flight
→ UPDATE bookings SET flight_status='completed', arrival_actual
→ INSERT INTO reviews
→ Trigger operator payout (+48h)

💡 تدفق Empty Leg

1️⃣ Operator/System detects empty return
→ INSERT INTO empty_legs (status='available', discount=40%)
2️⃣ AI matching finds eligible customers
→ Query users WHERE (location, history, preferences)
→ Calculate match_score for top 50
→ INSERT INTO notifications (×50)
3️⃣ Dutch auction (cron every 6h)
→ UPDATE empty_legs SET current_discount_pct += 10
→ Re-notify top matches
4️⃣ Customer books
→ UPDATE empty_legs SET status='sold'
→ INSERT INTO bookings (from empty_leg)
→ Fast payment flow

🏆 تدفق نظام الولاء

1️⃣ Booking paid
→ points = (total / 100) × multiplier
→ INSERT INTO loyalty_transactions (type='earn', points, multiplier)
→ UPDATE users SET loyalty_points += points, total_spent_yearly += amount
2️⃣ Tier upgrade check
→ IF total_spent_yearly crosses threshold:
UPDATE users SET loyalty_tier = new_tier
INSERT INTO notifications (type='loyalty')
3️⃣ Referral earn
→ Referred user completes first booking
→ INSERT INTO loyalty_transactions (user=referrer, type='referral', points=5000)
4️⃣ Points redemption
→ User redeems: 1000 points → 50 SAR credit
→ INSERT INTO loyalty_transactions (type='redeem', points=-1000)

🚁 تدفق MedEvac

1️⃣ Emergency request
→ INSERT INTO medevac_requests (status='received', severity)
→ Check subscription_id: IF valid, use covered_events
2️⃣ Dispatch (AI routing)
→ Find nearest aircraft WHERE is_medevac_capable = true
→ UPDATE medevac_requests SET status='dispatched'
3️⃣ In-transit tracking
→ UPDATE status='in_transit'
4️⃣ Delivery
→ UPDATE status='delivered', response_time_minutes, final_cost
→ IF subscription: UPDATE used_events += 1
→ ELSE: INSERT INTO bookings + payments
